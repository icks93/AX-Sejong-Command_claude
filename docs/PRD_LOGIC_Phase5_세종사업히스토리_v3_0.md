# PRD_LOGIC_Phase5_세종사업히스토리_v3.0.md
> 작성: 익스 + Claude | 2026.06.22
> 배경: V3 엑셀 개정 — 연간 "내부원가 현행화"(세종개발본부 전체 인건비 기준 매출원가 재산정) 반영을 위한 데이터 구조 변경

---

## 0. 한 줄 요약

연간 합계를 더 이상 웹앱이 프로젝트 행을 더해서 계산하지 않는다. **노션에 미리 계산된 "프로젝트합계"/"내부원가현행화" 두 행을 그대로 가져와서 보여준다.** 프로젝트별 상세 데이터는 새 DB(`_v3`)로 옮긴다.

---

## 1. 배경 — 왜 바뀌었나

기존(v2.x)에는 연간 합계를 웹앱이 그 해의 프로젝트 행들을 합산해서 만들었다. 이 방식의 매출원가는 "프로젝트별로 배분된 내부인건비"의 합이라, 프로젝트에 직접 배분되지 않는 인력(PM 관리, 지원업무 등)의 인건비가 빠진다.

V3 엑셀에서는 연간 단위로 **세종개발본부 전체 인건비**(`2.연도별조직및내부원가` 시트에서 산출)를 매출원가에 반영한 별도 행 "내부원가 현행화"를 만들었다. 그 결과 매출원가가 커지고 매출이익은 작아진다 — 이게 더 정확한 숫자다.

- `영업이익`/`영업이익률`은 두 유형(프로젝트합계/내부원가현행화)이 **동일한 값**을 가진다 — 이번 V3 개정에서는 매출원가/매출이익만 재산정했고 영업이익은 아직 갱신 전(판관비 데이터 미정비 상태와 연동, 추후 작업)

### 2.3 손익현황 DB — 변경 없음 (이미 처리 완료)

### 3.1 연도별 사업 현황 요약 — 데이터 소스 교체

- **기존(v2.x)**: 그 해 프로젝트 행들을 웹앱이 합산해서 매출/매출원가/매출이익 계산
- **변경(v3.0)**: `[AX]세종사업히스토리_연도별합계_v2`에서 해당 연도의 "내부원가현행화" 행을 가져와 그대로 표시 (재계산 금지)
- 같은 연도의 "프로젝트합계" 행도 같이 가져와서, 메인 숫자 아래 작은 글씨로 "(참고: 프로젝트합계 기준 OO.O억 / O.O억 / NN.N%)" 형태로 보조 표시
- 2020년은 "내부원가현행화" 행이 없으므로 "프로젝트합계" 행만 표시 (보조 표시 없음)
- **전년대비매출** 계산은 두 "내부원가현행화" 행(또는 2020년만 프로젝트합계) 간 비교로 계산 — 기준이 섞이지 않도록 주의

### 3.2 프로젝트별 상세 (연도 탭 클릭 시)

- **데이터 소스를 `[AX]세종사업히스토리_v2` → `[AX]세종사업히스토리_v3`로 교체**
- 표시 컬럼에 신규 4개(`내부인건비`/`외부인건비`/`외부SW`/`제경비기타`) 추가 여부는 익스 판단 필요 (기존 화면이 이미 충분히 컬럼이 많다면 상세 패널이나 토글로 빼는 것도 고려)
- 프로젝트 합계 행(기존엔 화면에 "계" 줄로 같이 나왔을 것)은 **이 표에서 빼고, 3.1절 연간 요약 블록에서만** 보여준다 (중복 제거)

### 3.3 v2.3 KPI 타일·캐시카우 배너 — 기준값 변경

- "매출이익률", "전년대비 매출이익률" 타일: **내부원가현행화 기준**으로 계산 (기존 프로젝트 합계 기준에서 변경 — 숫자가 작아짐에 유의, 예: 2026년 16.1%→8.4%)
- "수행조직 대비 매출 비중" 타일: 매출은 영향 없음(매출 자체는 안 바뀜), 비중 계산 그대로 유지
- 캐시카우 배너(운영매출 역대 최고): **영향 없음** — 운영매출(매출)은 이번 개정에서 바뀌지 않음, 매출원가/매출이익만 바뀜

## 4. API 연동

1. 기존 `/api/history` 엔드포인트가 두 신규 DB를 모두 조회하도록 확장
   - `[AX]세종사업히스토리_v3` → 프로젝트별 상세 (연도 탭용)
   - `[AX]세종사업히스토리_연도별합계_v2` → 연간 요약 (프로젝트합계 + 내부원가현행화, 연도별 매칭)
2. **단위 변환 필수**: 연도별합계 DB는 원 단위, 그래프/표 표시는 기존처럼 억원 단위이므로 ÷1e8 처리
3. 기존 "웹앱이 프로젝트 합산해서 연간 총계 계산하던 로직"은 **제거**하고 노션에서 가져온 값을 그대로 표시
4. 캐시 정책은 기존과 동일

## 6. 버전 변경 이력

| 버전 | 변경 |
|---|---|
| v2.3 | KPI 타일 5개 + 캐시카우 배너 추가 (이전 PRD) |
| **v3.0** | 데이터 소스를 `_v2`(프로젝트 DB만) → `_v3`(프로젝트) + `연도별합계_v2`(연간 요약) 2-DB 구조로 변경. 연간 합계를 웹앱 자체 계산 → 노션의 "내부원가현행화" 값 그대로 사용으로 전환. 프로젝트합계는 보조 표시로 유지 |

**v2.3 기준으로 이미 구현되어 있다면**: 연간 합계 계산 로직(프로젝트 행 합산하는 부분)을 찾아서 제거하고, 노션 조회로 교체하는 게 이번 작업의 핵심. KPI 타일 중 매출이익률 관련 숫자가 작아지는 건 버그가 아니라 의도된 변경.
