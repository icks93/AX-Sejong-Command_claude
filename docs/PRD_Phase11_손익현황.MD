## 1. 목적

세종개발본부가 회사(전사) 및 수행조직(SI·운영 등 돈 버는 개발조직) 대비 **인력 투입보다 매출·매출이익을 더 많이 만들어내고 있다**는 것을, 매년 데이터로 한눈에 보여주는 탭을 AX Sejong Command 웹앱에 추가한다.

핵심 메시지: "세종은 인력 비중보다 매출/매출원가/매출이익 비중이 항상 더 크다."

### 2.2 속성(컬럼) 목록

**원본 입력값 (Number, 수동 입력 — 엑셀 "연도별 손익현황" 시트 기준, 단위: 인력=명, 금액=억원)**

| 속성명 | 설명 |
|---|---|
| 연도 | Title. 예: "2026년" |
| 전사_인력 / 수행조직_인력 / 세종_인력 | 인력(명). 수행조직_인력 = 개발인력(연구소제외) |
| 전사_매출 / 수행조직_매출 / 세종_매출 | 매출(억원) |
| 전사_매출원가 / 수행조직_매출원가 / 세종_매출원가 | 매출원가(억원) |
| 전사_매출이익 / 수행조직_매출이익 / 세종_매출이익 | 매출이익(억원). **수행조직_매출이익은 2025년 음수(-8.45) 가능** |
| 전사_판관비 / 수행조직_판관비 / 세종_판관비 | 판관비(억원). **현재 전부 비어있음(추후 입력)** |
| 전사_영업이익 / 수행조직_영업이익 / 세종_영업이익 | 영업이익(억원). **현재 전부 비어있음(추후 입력)** |

**계산값 (Formula, 노션이 자동 계산 — 웹앱은 재계산하지 않고 그대로 표시)**

| 속성명 | 계산식(의미) |
|---|---|
| 인력비중_수행조직대비 / 인력비중_전사대비 | 세종_인력 ÷ 기준_인력 × 100 |
| 매출비중_수행조직대비 / 매출비중_전사대비 | 세종_매출 ÷ 기준_매출 × 100 |
| 매출원가비중_수행조직대비 | 세종_매출원가 ÷ 수행조직_매출원가 × 100 |
| 매출이익비중_수행조직대비 / 매출이익비중_전사대비 | 세종_매출이익 ÷ 기준_매출이익 × 100. **수행조직_매출이익<0인 해(2025)는 음수/이상값 — 2.3절 예외처리 참고** |
| 전사_매출이익률 / 세종_매출이익률 | 매출이익 ÷ 매출 × 100 |
| 전사_영업이익률 / 수행조직_영업이익률 / 세종_영업이익률 | 영업이익 ÷ 매출 × 100. 영업이익 미입력 상태라 현재 빈값 |
| 수행인력대비_매출_금액 | 세종_매출 − (수행조직_매출 × 세종_인력 ÷ 수행조직_인력) |
| 수행인력대비_매출_pp | 매출비중_수행조직대비 − 인력비중_수행조직대비 (%p) |
| 수행인력대비_매출원가_금액 / _pp | 매출원가 버전, 동일 패턴 |
| 수행인력대비_매출이익_금액 / _pp | 매출이익 버전, 동일 패턴. **2025년은 2.3절 예외처리 참고** |

### 2.3 예외 처리 — 2025년 (수행조직_매출이익 < 0)

2025년은 수행조직 전체가 매출이익 적자(-8.45억)였는데 세종은 흑자(7.53억)였던 해. 분모가 음수이므로 "비중(%)"과 "수행인력대비 초과분"이라는 개념 자체가 의미를 잃는다.

**규칙**: `수행조직_매출이익 < 0` 인 행에 대해서는, 매출이익 관련 비중·수행인력대비 컬럼 값을 화면에 그대로 숫자로 노출하지 말고 아래로 대체 표시한다.

- 미니차트(매출이익): 초과분 영역에 "수행조직 적자 / 속 세종 흑자" 텍스트 (2줄, 골드/레드)
- 연도별 효율 카드: 효율배수 자리에 "흑자전환 기여" 배지
- 요약표 "수행인력대비 매출이익" 셀: "흑자전환 기여" (골드, bold)

판단 조건(프론트/백엔드 공통): `해당연도.수행조직_매출이익 < 0`

## 4. 화면 구성 상세 (4블록)

> 색상·폰트·배경 등 비주얼 토큰은 익스가 기존 웹앱 캡처를 기반으로 별도 적용할 예정이므로, 이 PRD는 **레이아웃·데이터 매핑·인터랙션**만 명세한다. 참고용 색상/타이포는 `mockup_손익현황_v2.html` 그대로 따라가도 무방.

### 4.1 연도별 수행인력 효율배수 (카드 6개, 2021~2026)

- 가로 배치 카드 6개 (`grid-template-columns: repeat(auto-fit, minmax(165px,1fr))`)
- 카드당 표시 항목:
  - 연도
  - **효율배수** = 매출이익비중_수행조직대비 ÷ 인력비중_수행조직대비 (소수 2자리, "1.86x" 형식)
    - 2025년은 2.3절 예외처리에 따라 "흑자전환 기여" 텍스트로 대체
  - 인력 / 매출 / 매출원가 / 매출이익 4개 미니 바(각 항목의 `*비중_수행조직대비` 값을 길이로 표현)
    - 인력 바 위치에 점선 기준선 표시(= 인력비중_수행조직대비 자기 자신의 위치이므로, 다른 3개 바에 그려지는 기준선과 동일한 x 위치)
    - 각 바 우측에 `값.toFixed(1)%` 라벨
  - 2025년은 매출이익 바 생략하고 하단에 "수행조직 적자 속 세종 흑자 (하단 참고)" 1줄 표기

### 4.2 연도별 수행조직대비 매출·매출원가·매출이익 (미니차트 4개, 세로 1열 풀폭)

4개 차트: 인력 / 매출 / 매출원가 / 매출이익. 각 차트는 연도별(2021~2026) 그룹 바차트.

- 한 연도 그룹 = 수행조직 바(파란/스틸 톤) + 세종 바(골드 톤), 좌우 나란히
- **두 막대 모두 맨 위에 값 라벨 필수** (수행조직: `수행조직_X` 값, 세종: `세종_X` 값)
- 세종 바 라벨은 2줄: 1줄 = 금액(`세종_X.toFixed(1)+단위`), 2줄 = `(*비중_수행조직대비.toFixed(1)%)`
  - 단, 매출이익 차트의 2025년은 "(흑자)" 표기로 대체 (수행조직_매출이익<0)
- 인력 차트를 제외한 매출/매출원가/매출이익 차트에는:
  - 세종 바 위치에 **인력비중 기준선**(점선) 표시: `y(수행조직_X × 인력비중_수행조직대비 / 100)`
  - 점선 옆에 **초과분 삼각표 2줄**: 1줄 = `▲ |수행인력대비_X_금액|단위`, 2줄 = `(|수행인력대비_X_pp| %p)`
    - 증가(`수행인력대비_X_금액 ≥ 0`) = 빨간 ▲ / 감소 = 파란 ▼
    - 매출이익 차트의 2025년(수행조직_매출이익<0)은 삼각표 대신 "수행조직 적자 / 속 세종 흑자" 2줄 텍스트로 대체
- 인력 차트에는 기준선·초과분 표시 **하지 않음** (자기 자신이 기준이므로)

### 4.3 2025년 하이라이트 배너

- 고정 텍스트형 배너 (연도 동적 바인딩 가능하지만, 1차 구현은 2025년 하드코딩으로 충분)
- 표시 값: `수행조직_매출이익`(2025), `세종_매출이익`(2025), `수행조직_영업이익률`/`세종_매출이익률`(2025) — 영업이익률은 영업이익 입력 전까지 매출이익률로 대체 문구 사용
- "수행조직_매출이익 < 0인 가장 최근 연도"가 있으면 그 연도 기준으로 자동 노출, 없으면 배너 숨김 처리 (확장 여지로 명시, 1차는 2025 고정도 무방)

### 4.4 연도별 사업 현황 요약 (표)

- 연도별(2026→2021 역순) **2행 1세트** 구조: 위 = 수행조직, 아래 = 세종개발본부
- 컬럼: 연도(rowspan 2) / 구분 / 인력 / 매출 / 매출원가 / 매출이익 / 매출이익률 / 수행인력대비 매출 / 수행인력대비 매출원가 / 수행인력대비 매출이익
- 수행조직 행의 인력/매출/매출원가/매출이익 값은 비율 없이 단독 표기
- 세종 행의 인력/매출/매출원가/매출이익 값은 2줄: 1줄 금액, 2줄 `(*비중_수행조직대비%)` — 매출이익만 2025년 예외처리(흑자전환 텍스트)
- **표 정렬 규칙**: 값과 단위(억/명)를 같은 줄에, 비율은 그 아래 별도 줄로 분리하여 두 행(수행조직/세종)의 단위 위치가 시각적으로 정렬되도록 함 (각 td 내부를 div 2개로 분리, text-align:right 상속)
- 수행조직 행의 "수행인력대비" 3컬럼은 해당 개념이 적용 안 되므로 `colspan=3` 처리하여 "—" 한 줄로 표시
- 세종 행의 "수행인력대비" 3컬럼: `▲/▼ 금액단위(pp%p)` 형식, 매출이익만 수행조직_매출이익<0 해는 "흑자전환 기여"(골드) 표시
- 판관비/영업이익/영업이익률 컬럼은 **1차 구현에서는 비표시**. 추후 익스가 노션에 값 입력 후 요청 시 컬럼 추가(아래 6절 참고)

## 5. API 연동 방식

기존 Phase 5(세종사업히스토리) 연동 패턴과 동일하게 진행한다.

1. FastAPI 백엔드에 `/api/profit-loss` (또는 합의된 네이밍) 엔드포인트 추가
2. 백엔드가 Notion API로 `[AX]세종손익현황_연도별` 데이터소스를 쿼리, 6개 행(2021~2026)을 가져옴
3. Notion Formula 속성값은 **백엔드/프론트 어디서도 재계산하지 않고 그대로 사용** — 계산 책임은 노션에 있음
4. 프론트(Vanilla JS)는 받아온 JSON을 4절의 4개 블록에 매핑하여 렌더링
5. 캐싱 정책: 기존 Phase 5와 동일한 새로고침 주기 적용 (수동 새로고침 버튼 + 일정 주기 자동 갱신)

### 5.1 필드명 고정

웹앱 API 응답 필드명은 Notion 속성명을 그대로 사용한다 (한글 그대로, 언더스코어 구분). 별도 영문 alias가 필요하면 이 단계에서 백엔드가 매핑 테이블을 가지고 변환하되, **Notion 속성명 자체는 변경하지 않는다** (향후 다른 자동화에서도 재사용 가능하도록).

## 7. 비기능 요구사항

- 반응형: 미니차트 4개 블록은 데스크탑 기준 풀폭, 모바일에서는 가로 스크롤 또는 축소 비율 유지(SVG viewBox 방식이므로 자동 스케일됨)
