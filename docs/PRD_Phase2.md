## 3. KPI 박스 정의

| 박스 | ID | 색상 | 계산 방법 |
|------|-----|------|---------|
| 수행 사업 | `kpi-w-total` | 기본 | 파싱된 사업 섹션 수 |
| 이슈 | `kpi-w-issue` | 빨강 | 이슈 키워드 감지된 사업 수 |
| 위험 | `kpi-w-risk` | 주황 | 위험 키워드 감지된 사업 수 |
| 특이사항 | `kpi-w-special` | 파랑 | 특이사항 섹션 존재하는 사업 수 |
| 마감임박 | `kpi-w-deadline` | 노랑 | 7일 이내 마감 항목 있는 사업 수 |

## 7. 노션 블록 파싱 전략

노션 페이지는 블록 단위로 구성됩니다.

```python
# 주차 섹션 구분: heading_2 타입 + "[26-" 으로 시작
# 사업 섹션 구분: heading_3 타입 + "숫자. " 으로 시작
# 진행사항/특이사항/이슈사항: bold 텍스트 + bulleted_list_item
# 분석: "※ 분석" heading_3 이후 bulleted_list_item

블록 타입:
- heading_2  → 주차 구분선
- heading_3  → 사업명 섹션
- bulleted_list_item → 진행사항 각 항목
- paragraph  → 특이사항/이슈사항 내용
- divider    → 사업 구분선 (---)
```
