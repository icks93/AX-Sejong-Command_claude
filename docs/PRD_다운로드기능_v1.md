## 1. 메뉴별 다운로드 형식 (확정)

| # | 메뉴 | 형식 | 비고 |
|---|---|---|---|
| 1 | 대시보드 | 워드 | KPI 5개 + 4개 섹션 요약 |
| 2 | 사업현황 | 워드 | 표 + 그래프(매출 성장 추이) |
| 3 | 주간회의 | 워드 | 프로젝트별 진행사항/이슈사항 |
| 4 | 월간보고 | 워드 | AI전환 현황 + 프로젝트 현황 + 분석 |
| 5 | 월간분석 | **다운로드 없음** | 내부 전용, 화면으로만 |
| 6 | 사업히스토리 | 워드 | 운영매출 추이 차트 + 연도별 요약 + 프로젝트 상세 |
| 7 | 조직도 | **PDF** | 이미지/도형 구조라 워드보다 PDF가 적합 |
| 8 | 영업보고 | 워드 | 고객사별 접촉현황 + 파이프라인 |
| 9 | RISK 알림 | **다운로드 없음** | 슬랙·노션에 이미 누적되는 일회성 알림 |
| 10 | 세미나알림 | **다운로드 없음** | 슬랙·노션에 이미 누적되는 일회성 알림 |
| 11 | 손익현황 | 워드 | KPI 타일 + 효율배수 차트 + 요약표 |

**변환 주체**: 백엔드가 화면에 쓰는 데이터를 그대로 받아 서버에서 직접 파일 생성 (python-docx). 프론트는 "다운로드" 버튼만 누르면 됨.

## 2. 공통 보고서 디자인 템플릿 (모든 워드 문서 공통 적용)

지금까지 만든 사장님 보고서(`세종개발본부_현황히스토리손익_보고`)와 동일한 스타일을 **공통 템플릿 모듈**로 분리해서 8개 메뉴가 전부 재사용한다.

### 2.1 색상/폰트

| 요소 | 값 |
|---|---|
| 메인 색상(제목/구분선) | 네이비 `#1B2A4A` |
| 강조 색상(핵심 숫자/하이라이트) | 골드 `#B8860B` |
| 증가 표시 | 레드 `#C53030` |
| 감소/보조 표시 | 블루 `#2B6CB0` |
| 본문 텍스트 | `#1A1F2B` |
| 보조 텍스트(출처/각주) | 그레이 `#6B7280`, 이탤릭, 작은 폰트 |

### 2.2 공통 레이아웃 요소

- **헤더**: 우측 상단에 "{메뉴명} | YYYY.MM" 형식, 작은 그레이 텍스트
- **표지/제목**: 문서 최상단 중앙에 메뉴명(크게, 네이비, 굵게) + 부제(생성일·작성자, 작은 그레이)
- **섹션 제목(h1)**: 네이비 색, 하단에 네이비 밑줄
- **KPI 타일**: 표 형태로 가로 배치, 회색 배경 박스, 라벨(작게)+숫자(크게, 네이비 또는 골드)+보조설명(작게)
- **데이터 표**: 헤더 행은 네이비 배경+흰 글씨, 짝수 행은 옅은 회색 배경(zebra), 본문은 가운데 정렬(첫 컬럼은 좌측 정렬)
- **차트**: matplotlib으로 생성한 PNG를 본문 중앙에 삽입 (한글 폰트는 Noto Sans CJK KR 사용 필수 — 깨짐 방지)
- **각주**: 표/차트 아래 작은 회색 이탤릭으로 "출처: OOO" 표기
- **푸터**: 중앙에 "유라클 세종개발본부" 작게

### 2.3 구현 방식

`report_template.py` 같은 공통 모듈을 만들어서 아래 함수들을 8개 메뉴 생성 코드가 전부 공유:

```python

def make_kpi_table(doc, items):  # items = [(label, value, sub, color), ...]
def make_data_table(doc, headers, rows): ...
def embed_chart_image(doc, image_path, width=580): ...
def add_footnote(doc, text): ...
def setup_header_footer(doc, menu_name): ...
```

> 참고: 이미 손익현황 보고서(`build.js`는 docx 라이브러리, Node.js)와 다른 보고서들이 Python(python-docx)으로 만들어질 수도 있어 — **언어/라이브러리는 기존 백엔드 스택(FastAPI=Python)에 맞춰 python-docx로 통일하는 것을 권장.** Node.js의 docx 라이브러리로 만들었던 건 이번 작업과 별개(그쪽은 그대로 두고, 신규 8개는 Python 백엔드 안에서 생성).

## 3. 메뉴별 문서 구성

### 3.1 대시보드
- 표지: "대시보드 현황 보고"
- KPI 5개 (수행사업/총매출/팀인원/이번주영업/준비중) 타일
- 수행 사업 현황 표 (요약)
- 사업별 인력 현황 (표로 대체, 파이차트는 막대로 변환해 이미지 삽입)
- RISK 알림 상위 5건 + 주간회의 집중확인 + 영업보고 + 세미나알림 각 섹션 요약

### 3.2 사업현황
- 표지: "사업현황 보고"
- 매출 성장 추이 차트(이미지 삽입)
- 수주 완료 / 추가 수주 목표 표

### 3.3 주간회의
- 표지: "{N월N주차} 주간회의록"
- 프로젝트별(고정 5개+추가) 진행사항/특이사항/이슈사항
- 분석 섹션

### 3.4 월간보고
- 표지: "{N월} 세종개발본부 월간보고"
- AI전환 현황 / 프로젝트 진행 현황 / 인력투입현황(표) / 분석

### 3.5 사업히스토리
- 표지: "세종사업히스토리"
- 운영매출 추이 차트(이미지 삽입, 기존 화면 그래프와 동일)
- 연도별 사업 현황 요약 표 (내부원가현행화 기준 + 프로젝트합계 보조 표시)
- 선택 연도 프로젝트별 상세 표

### 3.6 영업보고
- 표지: "{N월N주차} 영업보고 분석"
- 고객사별 접촉 현황 표
- 세종본부 연관 기회 / 액션 필요 건 / 이슈·리스크

### 3.7 손익현황
- 기존에 만든 사장님 보고서와 동일 구성 (이미 완성된 템플릿 재사용)

### 3.8 조직도 (PDF, 워드 템플릿 미적용)
- 현재 화면에 표시되는 조직도 이미지를 그대로 PDF 1페이지로 변환
- 별도 디자인 템플릿 불필요 (이미지 그대로)

## 4. 구현 방식

1. 백엔드에 `/api/{menu}/download` 엔드포인트 8개 추가 (조직도는 `/api/org/download-pdf`)
2. 각 엔드포인트는 해당 메뉴의 기존 데이터 조회 함수를 재사용 (새로 노션 호출 만들지 않음)
3. `report_template.py` 공통 모듈로 문서 생성 후 `StreamingResponse` 또는 임시파일로 다운로드 응답
4. 프론트: 각 메뉴 화면 우측 상단에 "⬇ 다운로드" 버튼 1개 추가 (조직도는 "⬇ PDF 다운로드")
5. 조직도 PDF는 화면의 조직도 이미지/SVG를 서버에서 PDF로 변환 (예: 이미지라면 Pillow/reportlab으로 단순 페이지 구성)

## 5. 클로드 코드 프롬프트

```
AX Sejong Command에 메뉴별 다운로드 기능을 추가해줘.
PRD_다운로드기능_v1.md 참고.

1. 공통 모듈 report_template.py 먼저 만들어줘 (PRD §2 디자인 스펙대로)
   - make_heading1, make_kpi_table, make_data_table, embed_chart_image,
     add_footnote, setup_header_footer 함수
   - 색상: 네이비 #1B2A4A, 골드 #B8860B, 레드 #C53030, 블루 #2B6CB0
   - 차트는 matplotlib + Noto Sans CJK KR 폰트로 생성 (한글 깨짐 방지 필수)

2. 아래 8개 메뉴에 다운로드 버튼 + 백엔드 엔드포인트 추가
   - 대시보드, 사업현황, 주간회의, 월간보고, 사업히스토리, 영업보고, 손익현황 → 워드(.docx)
   - 조직도 → PDF
   - 월간분석/RISK알림/세미나알림은 다운로드 버튼 추가하지 마 (제외 확정)

3. 각 메뉴 문서 구성은 PRD §3 참고. 기존 화면에 이미 있는 데이터/차트를 그대로
   문서에 옮기는 것이 핵심 — 새로운 데이터 수집/계산 로직을 만들지 마

반드시 지켜야 할 것:
- 기존 화면(조회) 기능과 데이터 소스는 그대로, 다운로드 기능만 추가
- 문서 생성은 python-docx 사용 (백엔드가 FastAPI/Python이므로)
- 차트 한글 폰트 깨짐 검증 필수
- 8개 메뉴 디자인이 서로 다르게 보이면 안 됨 — 공통 템플릿 함수로 통일
- 애매한 부분(예: 어떤 데이터까지 문서에 넣을지)은 먼저 질문
```

## 6. 완료 후 확인 요청

- 8개 워드 문서 + 1개 PDF가 전부 같은 색상/폰트/스타일로 통일되어 보이는지
- 차트 삽입된 문서(사업현황/사업히스토리/손익현황)에서 한글이 깨지지 않는지
- 월간분석/RISK알림/세미나알림 화면에는 다운로드 버튼이 없는지
