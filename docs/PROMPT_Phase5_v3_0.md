## 변경 내용 (4개, 핵심은 1번)

### 1. 연간 합계 데이터 소스 전환 (가장 중요)
- **기존**: 그 해 프로젝트 행들을 웹앱이 직접 합산해서 연간 매출/매출원가/매출이익 계산
- **변경**: `[AX]세종사업히스토리_연도별합계_v2` DB에서 해당 연도의 "내부원가현행화" 행(유형=내부원가현행화)을 가져와 그대로 표시. **웹앱에서 합산하는 기존 로직은 제거**
- 이 DB는 **원 단위**다 (다른 곳과 달리 억원이 아님) — 화면 표시 전 ÷100,000,000 변환 필수
- 같은 연도의 "프로젝트합계"(유형=프로젝트합계) 행도 가져와서, 메인 숫자 아래 "(참고: 프로젝트합계 기준 OO.O억 / O.O억 / NN.N%)" 형태로 작게 보조 표시
- 2020년은 내부원가현행화 행이 없음(프로젝트합계만 존재) → 2020년은 보조 표시 없이 프로젝트합계만 메인으로 표시

### 2. 프로젝트별 상세 데이터 소스 교체
- `[AX]세종사업히스토리_v2` → `[AX]세종사업히스토리_v3`로 교체
- 프로젝트 목록에서 합계("계") 행은 빼고, 1번에서 새로 만든 연간 요약 블록에서만 합계를 보여줄 것 (중복 제거)

### 3. KPI 타일(v2.3에서 만든 것) 기준값 변경
- "매출이익률", "전년대비 매출이익률" 타일 계산을 **내부원가현행화 기준**으로 변경 (숫자가 작아짐, 정상임 — 예: 2026년 16.1%→8.4%)
- "수행조직 대비 매출 비중" 타일은 매출 자체가 안 바뀌므로 영향 없음
- 캐시카우 배너(역대 최고 운영매출)는 영향 없음 (매출원가/매출이익만 바뀌고 매출=운영매출은 그대로)

### 4. 단위 처리 검증
- `[AX]세종사업히스토리_연도별합계_v2`는 원 단위, `[AX]세종손익현황_연도별`(Phase 11)은 억원 단위로 서로 다르다. 혼동해서 가져다 쓰지 않도록 변환 지점에 주석으로 단위 명시할 것

## 반드시 지켜야 할 것
- Phase 1~4, 6~9 코드 절대 건드리지 마
- 기존 그래프(운영매출 추이)·연도 탭 UI 동작은 변경하지 말 것
- 한국어 주석 포함
- 애매한 판단(예: 프로젝트별 표에 신규 컬럼 4개를 보여줄지 여부)은 먼저 질문

## 완료 후 확인 요청
- 연간 요약의 매출원가/매출이익이 "내부원가현행화" 기준으로 나오는지 (2026년 예: 매출원가 46.3억, 매출이익 4.3억)
- 프로젝트합계 보조 표시가 같이 보이는지
- 2020년만 보조 표시 없이 단독으로 나오는지
- KPI 타일의 매출이익률이 줄어든 새 기준으로 갱신됐는지
